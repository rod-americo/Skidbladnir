# Papers

Use esta pasta apenas quando o projeto tiver pensamento científico associado ao software.

Finalidade desta pasta:

- registrar hipótese, método e desenho experimental
- explicitar métricas, critérios de avaliação e resultados
- separar discussão científica da documentação operacional em `docs/`

Regra prática:

- `docs/` documenta o sistema, sua operação e seus contratos
- `papers/` documenta como o problema é pensado, avaliado e justificado

Estrutura inicial sugerida:

- `0001-problema-e-hipotese.md`
- `0002-metodo-e-metricas.md`
- `0003-resultados-e-discussao.md`
